PROYECTO: Sincronizador de Archivos P2P
========================================

DESCRIPCIÓN GENERAL
-------------------
Este proyecto consiste en una aplicación de Python que permite la sincronización automática de archivos entre múltiples computadoras conectadas a una misma red local (LAN). Funciona bajo un modelo Peer-to-Peer (P2P), lo que elimina la necesidad de un servidor central; cada nodo actúa tanto como cliente como servidor.

PROPÓSITO
---------
El objetivo principal es mantener una carpeta compartida (`shared_folder`) sincronizada en tiempo real entre todos los dispositivos que ejecuten la aplicación. Es útil para compartir documentos, imágenes u otros archivos de manera rápida y sin configuración manual de direcciones IP.

FUNCIONAMIENTO TÉCNICO
----------------------
El sistema opera mediante tres componentes principales que trabajan en paralelo:

1. Descubrimiento de Nodos (Peer Discovery):
   - Utiliza sockets UDP para emitir mensajes de presencia ("HELLO") a toda la red local mediante Broadcast (Puerto 5000).
   - Mantiene una lista dinámica de pares activos, añadiendo nuevos nodos que responden y eliminando aquellos que dejan de emitir señales por más de 10 segundos.

2. Monitoreo de Archivos (Watchdog):
   - Emplea la librería `watchdog` para supervisar cambios en el directorio local `shared_folder`.
   - Detecta eventos de creación y modificación de archivos.
   - Al detectar un cambio, inicia automáticamente la transferencia del archivo modificado a todos los nodos conocidos en la lista de pares.

3. Transferencia de Archivos (File Transfer):
   - Utiliza sockets TCP (Puerto 5001) para la transmisión fiable de datos.
   - Protocolo de Transferencia:
     a. Envío de metadatos: Nombre del archivo, tamaño y Hash SHA-256.
     b. Verificación: El receptor comprueba si ya tiene el archivo y si el Hash coincide para evitar descargas redundantes.
     c. Transmisión: Envío del contenido del archivo en bloques (chunks).
     d. Integridad: Validación final mediante Hash SHA-256 para asegurar que el archivo no se corrompió.
   - Mecanismo de prevención de ecos: Evita que un archivo recibido sea re-enviado automáticamente al emisor original.

REQUISITOS
----------
- Python 3.x
- Librerías: `watchdog` (para monitoreo), `tkinter` (interfaz gráfica standar).
- Conexión a una red local (LAN) que permita tráfico UDP Broadcast y TCP.
